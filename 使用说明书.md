# 美股盯盘工具使用说明书

本文档说明当前美股盯盘工具的运行逻辑、告警规则、服务器维护方法和常见问题处理。

## 1. 项目目标

这个工具用于监控美股持仓的短时间价格波动。当某只股票在 3 / 5 / 10 分钟窗口内出现区间高低点异动时，程序会通过飞书个人机器人发送提醒。

当前已部署在香港服务器上，使用：

- 行情源：moomoo OpenD
- 告警通道：飞书个人机器人
- 运行方式：Ubuntu systemd 后台服务
- 监控周期：每 10 秒检查一次

## 2. 当前部署结构

服务器上的项目目录：

```bash
/opt/stock-watcher
```

moomoo OpenD 目录：

```bash
/opt/moomoo-opend/moomoo_OpenD_10.7.6708_Ubuntu18.04/moomoo_OpenD_10.7.6708_Ubuntu18.04
```

后台服务：

```bash
moomoo-opend.service
stock-watcher.service
```

服务关系：

```text
moomoo-opend.service 负责登录 moomoo 并提供本地行情端口 127.0.0.1:11111
stock-watcher.service 负责读取行情、计算波动、发送飞书提醒
```

## 3. 核心运行逻辑

程序每隔 `POLL_SECONDS` 秒从 moomoo OpenD 拉取一次股票价格。

当前配置：

```env
POLL_SECONDS=10
ALERT_WINDOWS=3:0.8|1.2|1.8:0.5,5:1|1.5|2|2.5:0.7,10:1.5|2.5|3.5:1.0
LADDER_COOLDOWN_SECONDS=60
SYMBOL_DIRECTION_COOLDOWN_SECONDS=180
```

含义是：

- 每 10 秒检查一次价格
- 每只股票同时维护 3 分钟、5 分钟、10 分钟三个滚动窗口
- 上涨判断：当前价格 vs 当前窗口内最低价
- 下跌判断：当前价格 vs 当前窗口内最高价
- 如果涨跌幅跨过新的阶梯，就发送飞书提醒
- 同一轮、同一股票、同一方向只发送最强的一条，避免刷屏

窗口是滚动窗口，不是固定分段。

示例：

```text
最近 5 分钟价格：100 -> 98 -> 99 -> 101
上涨判断：当前价 101 vs 窗口低点 98，涨幅 3.06%

最近 5 分钟价格：100 -> 102 -> 101 -> 99.5
下跌判断：当前价 99.5 vs 窗口高点 102，跌幅 2.45%
```

## 4. 告警策略

当前使用阶梯触发：

```text
3 分钟：0.8% / 1.2% / 1.8%
5 分钟：1% / 1.5% / 2% / 2.5%
10 分钟：1.5% / 2.5% / 3.5%
```

配置写在服务器：

```bash
/opt/stock-watcher/.env
```

对应配置：

```env
ALERT_WINDOWS=3:0.8|1.2|1.8:0.5,5:1|1.5|2|2.5:0.7,10:1.5|2.5|3.5:1.0
LADDER_COOLDOWN_SECONDS=60
```

格式说明：

```text
ALERT_WINDOWS：窗口分钟数:阶梯阈值列表:重置线
LADDER_COOLDOWN_SECONDS：同股票、同方向、同窗口、同阶梯的兜底冷却秒数
SYMBOL_DIRECTION_COOLDOWN_SECONDS：同股票、同方向的全局冷静期，用来减少多窗口连续刷屏
```

例如：

```env
5 分钟窗口上涨 1.05%：触发 5分钟 1% 阶梯
5 分钟窗口上涨 1.30%：不提醒，因为没有跨过新阶梯
5 分钟窗口上涨 1.55%：触发 5分钟 1.5% 阶梯
5 分钟窗口上涨 2.10%：触发 5分钟 2% 阶梯
5 分钟窗口上涨 0.65%：低于 0.7% 重置线，下一轮重新允许触发 5分钟 1% 阶梯
```

## 6. 交易时段说明

程序按照美东时间判断当前时段：

```text
pre-market：04:00 - 09:30
regular：09:30 - 16:00
after-hours：16:00 - 20:00
overnight：20:00 - 次日 04:00
closed：周五 20:00 - 周日 20:00
```

注意：

周末休市期间不会生成盘中阶梯提醒，避免 moomoo 的残留 overnight 字段触发假波动。

```text
行情时间(美东)
```

常规盘使用 moomoo 返回的行情时间；盘前、盘后、夜盘使用本程序抓取价格时的美东采样时间。这样可以避免夜盘价格已经更新，但 moomoo 的 `data_time` 仍停留在 16:00 的误导。

## 7. 修改监控股票

持仓文件：

```bash
/opt/stock-watcher/positions.json
```

编辑：

```bash
cd /opt/stock-watcher
nano positions.json
```

格式示例：

```json
[
  {
    "symbol": "AAPL",
    "name": "Apple",
    "enabled": true
  },
  {
    "symbol": "NVDA",
    "name": "NVIDIA",
    "enabled": true
  }
]
```

字段说明：

```text
symbol：股票代码，程序按这个拉取行情
name：飞书里显示的名称
enabled：是否启用
```

保存后检查 JSON：

```bash
python3 -m json.tool positions.json
```

重启 watcher：

```bash
systemctl restart stock-watcher
```

## 8. 修改告警策略

编辑配置：

```bash
cd /opt/stock-watcher
nano .env
```

找到：

```env
ALERT_WINDOWS=3:0.8|1.2|1.8:0.5,5:1|1.5|2|2.5:0.7,10:1.5|2.5|3.5:1.0
LADDER_COOLDOWN_SECONDS=60
```

常见修改：

```env
ALERT_WINDOWS=3:0.8|1.2|1.8:0.5,5:1|1.5|2|3:0.7,10:1.5|2.5|3.5:1.0
```

含义：

```text
把 5 分钟窗口最高阶梯从 2.5% 改为 3%
```

保存后重启：

```bash
systemctl restart stock-watcher
```

## 9. 服务管理命令

查看 watcher 状态：

```bash
systemctl status stock-watcher --no-pager
```

查看 OpenD 状态：

```bash
systemctl status moomoo-opend --no-pager
```

查看实时日志：

```bash
journalctl -u stock-watcher -f
```

查看最近日志：

```bash
journalctl -u stock-watcher -n 100 --no-pager
```

重启 watcher：

```bash
systemctl restart stock-watcher
```

重启 OpenD：

```bash
systemctl restart moomoo-opend
```

停止 watcher：

```bash
systemctl stop stock-watcher
```

启动 watcher：

```bash
systemctl start stock-watcher
```

确认服务是否正在运行：

```bash
systemctl is-active moomoo-opend stock-watcher
```

确认服务是否开机自启：

```bash
systemctl is-enabled moomoo-opend stock-watcher
```

正常结果：

```text
active
active
enabled
enabled
```

## 11. 临时测试低阈值

如果想测试飞书告警是否能触发，可以临时降低阈值。

编辑：

```bash
nano /opt/stock-watcher/.env
```

例如改成：

```env
ALERT_WINDOWS=3:0.03|0.1|0.2:0.01,5:0.03|0.1|0.2:0.01,10:0.05|0.1|0.2:0.02
```

重启：

```bash
systemctl restart stock-watcher
```

收到测试告警后，记得改回正式策略：

```env
ALERT_WINDOWS=3:0.8|1.2|1.8:0.5,5:1|1.5|2|2.5:0.7,10:1.5|2.5|3.5:1.0
```

然后再次重启：

```bash
systemctl restart stock-watcher
```

## 12. 常见问题

### 12.1 飞书没有收到消息

检查服务日志：

```bash
journalctl -u stock-watcher -n 100 --no-pager
```

重点看是否有：

```text
HTTP 400
Invalid ids
Missing required environment variable
```

如果是 `Invalid ids`，通常是 `FEISHU_RECEIVE_ID` 填错。

如果是缺环境变量，检查：

```bash
grep -n '^FEISHU_' /opt/stock-watcher/.env
```

### 12.2 行情拉不到

检查 OpenD 是否运行：

```bash
systemctl status moomoo-opend --no-pager
```

检查端口：

```bash
ss -ltnp | grep 11111
```

测试行情：

```bash
cd /opt/stock-watcher
.venv/bin/python watch.py --once
```

### 12.3 时段显示不对

当前代码按美东时间判断：

```text
regular / pre-market / after-hours / overnight
```

如果服务器时间是 UTC，这不影响判断，因为程序内部使用 `America/New_York` 时区。

### 12.4 修改配置后不生效

配置文件改完后必须重启：

```bash
systemctl restart stock-watcher
```

## 13. 安全注意事项

不要把以下内容公开发布：

```text
FEISHU_APP_SECRET
FEISHU_RECEIVE_ID
moomoo 登录账号
moomoo 登录密码
服务器 root 密码
```

飞书 `App Secret` 如果曾经发到聊天、截图、文档里，建议在飞书开发者后台重置。

OpenD 登录密码保存在：

```bash
/opt/moomoo-opend/moomoo_OpenD_10.7.6708_Ubuntu18.04/moomoo_OpenD_10.7.6708_Ubuntu18.04/OpenD.xml
```

建议不要把这个文件上传到公开仓库。
