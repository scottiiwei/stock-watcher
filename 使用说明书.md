# 美股盯盘工具使用说明书

本文档说明当前美股盯盘工具的运行逻辑、告警规则、服务器维护方法和常见问题处理。

## 1. 项目目标

这个工具用于监控美股持仓的短时间价格波动。当某只股票在最近 5 分钟窗口内涨跌超过设定阈值时，程序会通过飞书个人机器人发送提醒。

当前已部署在香港服务器上，使用：

- 行情源：moomoo OpenD
- 告警通道：飞书个人机器人
- 运行方式：Ubuntu systemd 后台服务
- 监控周期：每 10 秒检查一次

## 2. 当前部署结构

服务器上的项目目录：

```bash
/opt/stock-watcher
```

moomoo OpenD 目录：

```bash
/opt/moomoo-opend/moomoo_OpenD_10.7.6708_Ubuntu18.04/moomoo_OpenD_10.7.6708_Ubuntu18.04
```

后台服务：

```bash
moomoo-opend.service
stock-watcher.service
```

服务关系：

```text
moomoo-opend.service 负责登录 moomoo 并提供本地行情端口 127.0.0.1:11111
stock-watcher.service 负责读取行情、计算波动、发送飞书提醒
```

## 3. 核心运行逻辑

程序每隔 `POLL_SECONDS` 秒从 moomoo OpenD 拉取一次股票价格。

当前配置：

```env
POLL_SECONDS=10
PRICE_WINDOW_MINUTES=5
ALERT_RULES=normal:1:3,strong:2:3
```

含义是：

- 每 10 秒检查一次价格
- 每只股票维护最近 5 分钟的价格窗口
- 用当前价格和窗口内最早价格对比
- 如果涨跌幅达到策略阈值，就发送飞书提醒

窗口是滚动窗口，不是固定分段。

示例：

```text
第 0 分钟：100
第 1 分钟：101，涨幅 1%，触发普通异动
第 6 分钟：104，此时第 0 分钟价格已过期，会用第 1 分钟的 101 对比
```

## 4. 告警策略

当前有两档策略：

```text
普通异动：5 分钟内涨跌超过 1%，冷却 3 分钟
强异动：5 分钟内涨跌超过 2%，冷却 3 分钟
```

配置写在服务器：

```bash
/opt/stock-watcher/.env
```

对应配置：

```env
ALERT_RULES=normal:1:3,strong:2:3
```

格式说明：

```text
策略ID:阈值百分比:冷却分钟
```

例如：

```env
ALERT_RULES=normal:1:3,strong:2:3
```

表示：

```text
normal 策略，阈值 1%，冷却 3 分钟
strong 策略，阈值 2%，冷却 3 分钟
```

两档策略的冷却时间彼此独立。

示例：

```text
第 1 分钟上涨 1.1%，触发普通异动
第 2 分钟上涨 2.2%，仍然可以触发强异动
第 3 分钟上涨 2.4%，强异动仍在 3 分钟冷却期内，不重复提醒
```

## 5. 飞书消息格式

启动消息示例：

```text
✅ 美股盯盘已启动
👀 监控: AAPL, NVDA, MU, MRVL, VOO
🪟 窗口: 5 分钟
🎯 策略: 普通异动 1% / 冷却 3 分钟
🎯 策略: 强异动 2% / 冷却 3 分钟
```

普通异动示例：

```text
⚡ 美股盯盘提醒｜普通异动
🏷️ 标的: NVDA NVIDIA
📈 波动: 1.0 分钟内上涨 1.20%
💵 价格: 100.00 -> 101.20
⏱️ 时段: regular
🕒 行情时间(美东): 2026-06-05 10:30:00
```

强异动示例：

```text
🔥 美股盯盘提醒｜强异动
🏷️ 标的: AAPL Apple
📉 波动: 2.0 分钟内下跌 2.05%
💵 价格: 102.00 -> 99.91
⏱️ 时段: regular
🕒 行情时间(美东): 2026-06-05 10:30:00
```

## 6. 交易时段说明

程序按照美东时间判断当前时段：

```text
pre-market：04:00 - 09:30
regular：09:30 - 16:00
after-hours：16:00 - 20:00
overnight：20:00 - 次日 04:00
```

注意：

```text
行情时间(美东)
```

表示 moomoo 返回的行情时间，不是飞书消息发送时间。

## 7. 修改监控股票

持仓文件：

```bash
/opt/stock-watcher/positions.json
```

编辑：

```bash
cd /opt/stock-watcher
nano positions.json
```

格式示例：

```json
[
  {
    "symbol": "AAPL",
    "name": "Apple",
    "enabled": true
  },
  {
    "symbol": "NVDA",
    "name": "NVIDIA",
    "enabled": true
  }
]
```

字段说明：

```text
symbol：股票代码，程序按这个拉取行情
name：飞书里显示的名称
enabled：是否启用
```

保存后检查 JSON：

```bash
python3 -m json.tool positions.json
```

重启 watcher：

```bash
systemctl restart stock-watcher
```

## 8. 修改告警策略

编辑配置：

```bash
cd /opt/stock-watcher
nano .env
```

找到：

```env
ALERT_RULES=normal:1:3,strong:2:3
```

常见修改：

```env
ALERT_RULES=normal:1:2,strong:2:3
```

含义：

```text
普通异动 1%，冷却 2 分钟
强异动 2%，冷却 3 分钟
```

保存后重启：

```bash
systemctl restart stock-watcher
```

## 9. 服务管理命令

查看 watcher 状态：

```bash
systemctl status stock-watcher --no-pager
```

查看 OpenD 状态：

```bash
systemctl status moomoo-opend --no-pager
```

查看实时日志：

```bash
journalctl -u stock-watcher -f
```

查看最近日志：

```bash
journalctl -u stock-watcher -n 100 --no-pager
```

重启 watcher：

```bash
systemctl restart stock-watcher
```

重启 OpenD：

```bash
systemctl restart moomoo-opend
```

停止 watcher：

```bash
systemctl stop stock-watcher
```

启动 watcher：

```bash
systemctl start stock-watcher
```

确认服务是否正在运行：

```bash
systemctl is-active moomoo-opend stock-watcher
```

确认服务是否开机自启：

```bash
systemctl is-enabled moomoo-opend stock-watcher
```

正常结果：

```text
active
active
enabled
enabled
```

## 10. 手动测试

测试 OpenD 是否监听：

```bash
ss -ltnp | grep 11111
```

正常应看到：

```text
127.0.0.1:11111
```

测试拉取一次行情：

```bash
cd /opt/stock-watcher
.venv/bin/python watch.py --once
```

正常输出示例：

```text
AAPL 311.3499 13:50:26.722 regular
NVDA 219.3050 13:50:26.731 regular
```

前台运行 watcher：

```bash
cd /opt/stock-watcher
.venv/bin/python watch.py
```

停止前台运行：

```text
Ctrl + C
```

## 11. 临时测试低阈值

如果想测试飞书告警是否能触发，可以临时降低阈值。

编辑：

```bash
nano /opt/stock-watcher/.env
```

例如改成：

```env
ALERT_RULES=normal:0.03:3,strong:0.1:3
```

重启：

```bash
systemctl restart stock-watcher
```

收到测试告警后，记得改回正式策略：

```env
ALERT_RULES=normal:1:3,strong:2:3
```

然后再次重启：

```bash
systemctl restart stock-watcher
```

## 12. 常见问题

### 12.1 飞书没有收到消息

检查服务日志：

```bash
journalctl -u stock-watcher -n 100 --no-pager
```

重点看是否有：

```text
HTTP 400
Invalid ids
Missing required environment variable
```

如果是 `Invalid ids`，通常是 `FEISHU_RECEIVE_ID` 填错。

如果是缺环境变量，检查：

```bash
grep -n '^FEISHU_' /opt/stock-watcher/.env
```

### 12.2 行情拉不到

检查 OpenD 是否运行：

```bash
systemctl status moomoo-opend --no-pager
```

检查端口：

```bash
ss -ltnp | grep 11111
```

测试行情：

```bash
cd /opt/stock-watcher
.venv/bin/python watch.py --once
```

### 12.3 时段显示不对

当前代码按美东时间判断：

```text
regular / pre-market / after-hours / overnight
```

如果服务器时间是 UTC，这不影响判断，因为程序内部使用 `America/New_York` 时区。

### 12.4 修改配置后不生效

配置文件改完后必须重启：

```bash
systemctl restart stock-watcher
```

## 13. 安全注意事项

不要把以下内容公开发布：

```text
FEISHU_APP_SECRET
FEISHU_RECEIVE_ID
moomoo 登录账号
moomoo 登录密码
服务器 root 密码
```

飞书 `App Secret` 如果曾经发到聊天、截图、文档里，建议在飞书开发者后台重置。

OpenD 登录密码保存在：

```bash
/opt/moomoo-opend/moomoo_OpenD_10.7.6708_Ubuntu18.04/moomoo_OpenD_10.7.6708_Ubuntu18.04/OpenD.xml
```

建议不要把这个文件上传到公开仓库。
