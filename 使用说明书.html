美股盯盘工具

监控美股持仓在短时间内的价格波动。当股票在最近 5 分钟滚动窗口内达到设定涨跌幅时，程序会通过飞书个人机器人发送告警。

行情源moomoo OpenD
通知通道飞书机器人
检查频率10 秒
正式策略1% / 2%

2. 当前部署结构

服务器上的项目目录：

/opt/stock-watcher

moomoo OpenD 目录：

/opt/moomoo-opend/moomoo_OpenD_10.7.6708_Ubuntu18.04/moomoo_OpenD_10.7.6708_Ubuntu18.04
服务作用状态
moomoo-opend.service登录 moomoo，提供本地行情端口 127.0.0.1:11111后台运行，开机自启
stock-watcher.service读取行情、计算波动、发送飞书提醒后台运行，开机自启

3. 核心运行逻辑

1. 读取持仓从 positions.json 读取要监控的股票代码。
2. 拉取行情每 10 秒通过 OpenD 获取最新价格。
3. 滚动计算维护最近 5 分钟价格窗口，计算涨跌幅。
4. 发送提醒达到策略阈值后，通过飞书机器人发送告警。

当前核心配置：

POLL_SECONDS=10
PRICE_WINDOW_MINUTES=5
ALERT_RULES=normal:1:3,strong:2:3
滚动窗口说明：窗口不是固定分段，而是每次报价到来时，都取“当前时间往前 5 分钟内”的最早价格进行比较。 
第 0 分钟：100
第 1 分钟：101，涨幅 1%，触发普通异动
第 6 分钟：104，第 0 分钟价格已过期，会用第 1 分钟的 101 对比

4. 告警策略

级别规则冷却说明
⚡ 普通异动5 分钟内涨跌超过 1%3 分钟提示股票开始出现明显波动。
🔥 强异动5 分钟内涨跌超过 2%3 分钟提示股票正在剧烈波动，需要更高关注。

配置文件位置：

/opt/stock-watcher/.env

正式配置：

ALERT_RULES=normal:1:3,strong:2:3

格式：

策略ID:阈值百分比:冷却分钟
两档策略的冷却时间彼此独立。普通异动刚提醒过，如果价格继续冲到强异动阈值，强异动仍然可以单独提醒。

6. 交易时段说明

时段美东时间说明
pre-market04:00 - 09:30盘前
regular09:30 - 16:00正常盘
after-hours16:00 - 20:00盘后
overnight20:00 - 次日 04:00夜盘
行情时间(美东) 是 moomoo 返回的行情时间，不是飞书消息发送时间。

7. 修改监控股票

持仓文件：

/opt/stock-watcher/positions.json

编辑命令：

cd /opt/stock-watcher
nano positions.json

格式示例：

[
  {
    "symbol": "AAPL",
    "name": "Apple",
    "enabled": true
  },
  {
    "symbol": "NVDA",
    "name": "NVIDIA",
    "enabled": true
  }
]
字段含义
symbol股票代码，程序按这个字段拉取行情。
name飞书消息里显示的名称。
enabled是否启用。设为 false 可临时停用某只股票。

保存后检查 JSON：

python3 -m json.tool positions.json

重启 watcher：

systemctl restart stock-watcher

8. 修改告警策略

编辑配置：

cd /opt/stock-watcher
nano .env

找到：

ALERT_RULES=normal:1:3,strong:2:3

示例：把普通异动冷却改成 2 分钟。

ALERT_RULES=normal:1:2,strong:2:3

保存后重启：

systemctl restart stock-watcher

9. 服务管理命令

目的命令
查看 watcher 状态systemctl status stock-watcher --no-pager
查看 OpenD 状态systemctl status moomoo-opend --no-pager
查看实时日志journalctl -u stock-watcher -f
查看最近日志journalctl -u stock-watcher -n 100 --no-pager
重启 watchersystemctl restart stock-watcher
重启 OpenDsystemctl restart moomoo-opend
确认运行状态systemctl is-active moomoo-opend stock-watcher
确认开机自启systemctl is-enabled moomoo-opend stock-watcher

正常健康状态：

active
active
enabled
enabled

10. 手动测试

测试 OpenD 是否监听

ss -ltnp | grep 11111

正常应看到：

127.0.0.1:11111

测试拉取一次行情

cd /opt/stock-watcher
.venv/bin/python watch.py --once

正常输出示例：

AAPL 311.3499 13:50:26.722 regular
NVDA 219.3050 13:50:26.731 regular

临时降低阈值测试飞书提醒

ALERT_RULES=normal:0.03:3,strong:0.1:3
测试完成后记得改回正式策略：ALERT_RULES=normal:1:3,strong:2:3，然后执行 systemctl restart stock-watcher。

11. 常见问题与排错

飞书没有收到消息

journalctl -u stock-watcher -n 100 --no-pager
grep -n '^FEISHU_' /opt/stock-watcher/.env

如果日志里出现 Invalid ids，通常是 FEISHU_RECEIVE_ID 填错。

行情拉不到

systemctl status moomoo-opend --no-pager
ss -ltnp | grep 11111
cd /opt/stock-watcher
.venv/bin/python watch.py --once

修改配置后不生效

systemctl restart stock-watcher
