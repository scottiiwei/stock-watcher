美股盯盘工具

监控美股持仓在短时间内的价格波动。当股票在 3 / 5 / 10 分钟窗口内出现区间高低点异动时，程序会通过飞书个人机器人发送告警。

行情源moomoo OpenD
通知通道飞书机器人
检查频率10 秒
正式策略3/5/10 分钟多窗口

2. 当前部署结构

服务器上的项目目录：

/opt/stock-watcher

moomoo OpenD 目录：

/opt/moomoo-opend/moomoo_OpenD_10.7.6708_Ubuntu18.04/moomoo_OpenD_10.7.6708_Ubuntu18.04
服务作用状态
moomoo-opend.service登录 moomoo，提供本地行情端口 127.0.0.1:11111后台运行，开机自启
stock-watcher.service读取行情、计算波动、发送飞书提醒后台运行，开机自启

3. 核心运行逻辑

1. 读取持仓从 positions.json 读取要监控的股票代码。
2. 拉取行情每 10 秒通过 OpenD 获取最新价格。
3. 滚动计算维护 3 / 5 / 10 分钟窗口，按区间低点/高点计算涨跌幅。
4. 发送提醒达到策略阈值后，通过飞书机器人发送告警。

当前核心配置：

POLL_SECONDS=10
ALERT_WINDOWS=3:0.8|1.2|1.8:0.5,5:1|1.5|2|2.5:0.7,10:1.5|2.5|3.5:1.0
LADDER_COOLDOWN_SECONDS=60
SYMBOL_DIRECTION_COOLDOWN_SECONDS=180
滚动窗口说明：窗口不是固定分段。上涨用“当前价 vs 窗口内最低价”，下跌用“当前价 vs 窗口内最高价”，因此能识别先砸后拉、先拉后砸这类异动。 
最近 5 分钟：100 -> 98 -> 99 -> 101
上涨判断：101 vs 低点 98，涨幅 3.06%

最近 5 分钟：100 -> 102 -> 101 -> 99.5
下跌判断：99.5 vs 高点 102，跌幅 2.45%

4. 告警策略

级别规则冷却说明
⚡ 3分钟0.8% / 1.2% / 1.8%60 秒兜底捕捉短线急动。
📣 5分钟1% / 1.5% / 2% / 2.5%60 秒兜底主力盯盘窗口。
🔥 10分钟1.5% / 2.5% / 3.5%60 秒兜底识别趋势扩散。
🧹 同轮去重同股票 + 同方向只发最强一条-避免多个窗口同时刷屏。

配置文件位置：

/opt/stock-watcher/.env

正式配置：

ALERT_WINDOWS=3:0.8|1.2|1.8:0.5,5:1|1.5|2|2.5:0.7,10:1.5|2.5|3.5:1.0
LADDER_COOLDOWN_SECONDS=60

格式：

窗口分钟数:阶梯阈值|阶梯阈值|阶梯阈值:重置线
阶梯状态按“股票 + 方向 + 窗口”记录。跨过更高阶梯才会再次提醒；回落到该窗口重置线以下后，下一轮可以重新触发。

6. 交易时段说明

时段美东时间说明
pre-market04:00 - 09:30盘前
regular09:30 - 16:00正常盘
after-hours16:00 - 20:00盘后
overnight20:00 - 次日 04:00夜盘
closed周五 20:00 - 周日 20:00周末休市，不发送盘中阶梯提醒
周末休市期间不会生成盘中阶梯提醒，避免 moomoo 的残留 overnight 字段触发假波动。
行情时间(美东)：常规盘使用 moomoo 返回的行情时间；盘前、盘后、夜盘使用本程序抓取价格时的美东采样时间。这样可以避免夜盘价格已经更新，但 moomoo 的 data_time 仍停留在 16:00 的误导。

7. 修改监控股票

持仓文件：

/opt/stock-watcher/positions.json

编辑命令：

cd /opt/stock-watcher
nano positions.json

格式示例：

[
  {
    "symbol": "AAPL",
    "name": "Apple",
    "enabled": true
  },
  {
    "symbol": "NVDA",
    "name": "NVIDIA",
    "enabled": true
  }
]
字段含义
symbol股票代码，程序按这个字段拉取行情。
name飞书消息里显示的名称。
enabled是否启用。设为 false 可临时停用某只股票。

保存后检查 JSON：

python3 -m json.tool positions.json

重启 watcher：

systemctl restart stock-watcher

8. 修改告警策略

编辑配置：

cd /opt/stock-watcher
nano .env

找到：

ALERT_WINDOWS=3:0.8|1.2|1.8:0.5,5:1|1.5|2|2.5:0.7,10:1.5|2.5|3.5:1.0
LADDER_COOLDOWN_SECONDS=60

示例：把 5 分钟窗口最高阶梯从 2.5% 改成 3%。

ALERT_WINDOWS=3:0.8|1.2|1.8:0.5,5:1|1.5|2|3:0.7,10:1.5|2.5|3.5:1.0

保存后重启：

systemctl restart stock-watcher

10. 手动测试

测试 OpenD 是否监听

ss -ltnp | grep 11111

正常应看到：

127.0.0.1:11111

测试拉取一次行情

cd /opt/stock-watcher
.venv/bin/python watch.py --once

正常输出示例：

AAPL 311.3499 13:50:26.722 regular
NVDA 219.3050 13:50:26.731 regular

临时降低阈值测试飞书提醒

ALERT_WINDOWS=3:0.03|0.1|0.2:0.01,5:0.03|0.1|0.2:0.01,10:0.05|0.1|0.2:0.02
测试完成后记得改回正式策略：ALERT_WINDOWS=3:0.8|1.2|1.8:0.5,5:1|1.5|2|2.5:0.7,10:1.5|2.5|3.5:1.0，然后执行 systemctl restart stock-watcher。

11. 常见问题与排错

飞书没有收到消息

journalctl -u stock-watcher -n 100 --no-pager
grep -n '^FEISHU_' /opt/stock-watcher/.env

如果日志里出现 Invalid ids，通常是 FEISHU_RECEIVE_ID 填错。

行情拉不到

systemctl status moomoo-opend --no-pager
ss -ltnp | grep 11111
cd /opt/stock-watcher
.venv/bin/python watch.py --once

修改配置后不生效

systemctl restart stock-watcher
